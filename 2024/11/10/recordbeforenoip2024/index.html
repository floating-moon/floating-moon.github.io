发表于 2024-11-10 更新于 2025-01-01 分类于 笔记

止颓暂卷。

11.10

炼石NOIP19 C

link

观察到贡献是 \(\sum \min(pre_i, suf_i) - a_i\)，考虑怎么刻画前后最大值的较小值，发现是建出大根笛卡尔树后从根到 \(i\) 路径上的第一个拐点，于是考虑从大到小插入笛卡尔树，假设我们现在要插入 \(a_i\)，我们就可以找到一个 \(a_j > a_i\)，造成 \(a_j - a_i\) 的贡献，另一个重要的观察是任意的 \(a_j - a_i\) 都是合法的，首先若 \(j\) 在笛卡尔树的左链或右链上，满足条件，否则设根到 \(j\) 的第一个拐点为 \(k\)，我们把 \(j\) 挂到 \(k\) 的左/右链上，再将 \(j\) 的两棵子树合并到 \(j\) 的父亲上，这样贡献减少了 \(a_k - a_j\)，我们再把 \(i\) 插进 \(k\) 的子树，这样贡献加上 \(a_k - a_i\)，所以总贡献还是 \(a_j - a_i\)，于是直接 bitset 优化背包。

炼石NOIP19 D

link

赛时被 LJ07 喂了做法。

考虑 A 性质，显然是问一次两者的值域都减半，于是写个主席树上二分即可。

然后我们考虑拓展，查一次 \([l_1, r_1]\) 最大值不合法，会将 \([l_2, r_2]\) 可选范围减半，我们为了平衡两边的值域，可以查一次 \([l_2, r_2]\) 的最大值是否合法，这样会将 \([l_1, r_1]\) 的值域变成和 \([l_2, r_2]\) 一样，这样用 \(2\) 次操作值域减半，就可以过了。

LG10743

link

或起来的一定是值域上的前缀，且起来的一定是值域上的一段后缀，且不同的前缀或/后缀与只有 \(O(\log V)\) 段，我们考虑把这些连续段拉出来，判断区间相交即可，拉出连续段可以通过开 \(\log V\) 个 ST 表来查询下一个影响它的最小值/最大值，注意可能存在相同元素，需要进行一些细节处理（比如把值相同的元素分开编号），然后就 \(O(n\log^2 V)\) 了。

11.11

NFLS C

link

min-max 容斥的套路还是见少了，以后多见一些，还有就是多看具体数学。

答案显然是 \(E(\max S)\) 的形式，我们转换为求 \(E(\min S)\)，然后这个东西显然只和 \(|S|\) 有关，我们记 \(f(x)\) 表示 \(|S| = x\) 时 \(E(\min S)\) 的值，再设一个 \(g(x)\) 表示操作都选在一个大小为 \(x\) 为的集合中的期望次数，那么有 \[ \begin{aligned} g(x) &= \sum_{i = 1}^x \dfrac{x^{\underline{i}}}{(n + m)^{\underline{i}}} \\ &= \sum_{i = 1}^x \dfrac{\binom{x}{i}}{\binom{n + m}{i}} \\ &= \sum_{i = 1}^x \dfrac{\binom{n + m - i}{n + m - x}}{\binom{n + m}{x}} \\ &= \dfrac{x}{n + m - x + 1} \end{aligned} \] 其中用到了若干具体数学第五章的奇技淫巧，然后我们考虑怎么算 \(f(x)\)，其实就是我们先若干次操作取在了大小为 \(n - x\) 的补集中，然后取了一张 Z 牌或一张我们需要的牌，如果取到 Z 牌的话需要重算，反之则停止，所以这一过程一共期望进行 \(\dfrac{x + m}{x}\) 轮，于是有 \(f(x) = (g(n - x) + 1)\dfrac{x + m}{x}\)，然后再套一个 min-max 容斥的式子就做完了。

NFLS D

link

感觉就是一道很简单的题，赛时想想就会了。

如果你做过超级钢琴的话可以获得 \(50\) 分，然后你会 \(k = 1\) 的话就多了 \(15\) 分。

考虑我们直接做超级钢琴的复杂度瓶颈在哪，我们一开始往堆中 push 了 \(r - l + 1\) 个元素，导致这一部分的复杂度为 \(O(nm\log n)\)，但是我们发现这是不必要的，因为我们肯定只关心答案最大的 \(k\) 个左端点，所以我们套用 \(k = 1\) 的做法，每次取出答案最大的左端点，然后把这个左端点 ban 掉，重复 \(k\) 次即可，ban 左端点即将这个点的 \(\max\) 设为 \(-\infty\)，然后这个时候再用之前的做法就行了。

11.13

炼石NOIP20 C

link

最唐的一集.jpg

场上降智严重，不写好写的 \(O(m^22^m)\) 跑去写 \(O(3^m)\)，简直非常抽象。

其实也不难，考虑转成概率去做，令 \(f(S)\) 表示到达 \(S\) 集合的概率，因为一个集合如果重复选两次不会造成任何贡献，所以我们有 \(\dfrac{f(S)}{c(S)} \to f(S \cap S_i)\)，其中 \(c(S)\) 为满足 \(S_i \cap S \neq \varnothing, S\) 的 \(S_i\) 个数，可以通过两遍高维前缀和求出，考虑令 \(g(S)\) 表示集合 \(S\) 的出现次数，那么就有 \(\dfrac{f(S)}{c(S)}g(T) \to f(S \cap T)\)，是一个 AND 卷积的形式，按照 popcount 从大到小转移即可做到 \(O(m^22^m)\)。

FXT 模拟赛 C

link

完全没理解题解在说啥，讲下赛时做法。

考虑 \(p = q^{-1}\)，那么其实就是要将 \(q\) 排序，那么我们假设 \(q\) 有 \(k\) 个上升段，我们就从 \(k/2\) 个上升段结尾处劈一刀，然后将 第 \(i\) 段和第 \(i + k\) 段归并起来，这样每次上升段个数减半，可以在 \(\lceil \log_2 k \rceil\) 次操作做完，不难证明每次上升段最多减半，所以这是最优解。

下午和 lyy 没事干 vp 了 LNCPC，两个人打了两个小时写了 11 个题，其中我写了 6 个，一人有一题的首A，感觉打 ACM 调节一下还是非常开心的。

LNCPC H

link

一眼看上去不是很好做，但是发现 \(l, r\) 实在很小，所以可以直接枚举数字和，记 \(f(p, s, t)\) 表示钦定还有 \(p\) 位没填，钦定数字和为 \(s\)，当前填的前缀和为 \(t\) 的答案，我们显然是在第一次 \(t > s / 2\) 的时候对答案进行贡献，系数为后面填数的方案，再做一遍类似的 DP 即可。

11.15

ZJ模拟赛 C

最唐的一集2.0.jpg

赛时写出来了正解，但是因为脑抽写了神秘 abs 导致挂了 \(30\) 分，以后能上拍还是尽量上拍，注意极限数据测 fsanitize 来避免 RE。

这个东西看一眼就很区间 DP，但是由于贡献和猜的数有关所以我们并不是很好直接去算，我们需要一个好计算贡献的方式，假设我们现在知道 \(k \in [L, R]\)，猜了一个数 \(u\) 把范围缩成 \([L, x]\) 与 \([x + 1, R]\)，那么如果答案在左边会贡献一个 \(u - a_k\)，在右边会贡献一个 \(a_k - u\)，其中 \(u\) 对答案的贡献是好算的，我们对 \(a_k\) 的部分考虑费用提前计算，也就是递归进入两侧的时候直接算好 \(a_k\) 的贡献，观察到我们递归进入左侧时相当于给答案中 \(a_k\) 的贡献系数 \(-1\)，进入右侧相当于给 \(a_k\) 的贡献系数 \(+1\)，那么我们就可以设 \(f(L, R, c)\) 表示已知 \(k \in [L, R]\)，且每个位置需要额外加上 \(c \times a_i\) 贡献时的答案，那么我们有转移 \[ f(L, R, c) = \min_x\{\max(f(L, x, c - 1) + g(x), f(x + 1, R, c + 1) - g(x))\} \] 其中边界状态 \(f(i, i, c) = c\times a_i\)

其中 \(g(x)\) 表示将区间划分为以 \(x\) 为分界的左右两部分的最优转移点，容易 \(O(1)\) 求出。

然后我们感受一下最优解的形态，发现 \(c\) 这一维大概是 \(O(\log V)\) 的，再感受一下，又发现 \(f(L, R)\) 在转移点从左往右移动时大概是先递减后递增，于是可以直接二分谷底位置，这样就做到 \(O(n^2\log n\log V)\)，因为写的是记搜，很多状态不会到，所以跑的比一只 \(\log\) 还快。

CCPC CQ A

link

算是一个经典套路，考虑小于 \(\sqrt{3000}\) 的质数只有 \(16\) 个，于是可以直接对每个小质数做状压 DP，然后再按照大质数分组合并，但这样合并复杂度有点小寄，于是考虑直接每组插入数字，贡献只与当前大质数是否出现过有关，于是记两个 DP 数组转移就行了。

1123模拟赛 B

赛时脑抽想了两小时容斥，以后记住一个思路想大概半小时不出来就要换一个了。

考虑直接 DP，记 \(f(i, j, k)\) 表示在 \(i\) 子树中已经确定了 \(j\) 条路径，是否向上延申的方案数，转移就是可以不选 \(i\)，也可以从子树中选出 \(\le 2\) 条向上延申的链拼在一起，或是选出 $ $ 条继续向上延申，是个类似背包的形式，于是就做完了。

1123模拟赛 C

没见过的 trick

记住树上路径修改可以尝试想把重儿子和轻儿子分开维护，这样可以用树剖加速。

1121模拟赛 D

Border 有一个性质就是能被划分为 \(\log n\) 段等差数列，所以我们可以在失配树上做类似树剖的东西，如果 \(fa_{fa_i}, fa_i, i\) 构成等差数列则称 \(fa_i \to i\) 为重边，反之为轻边，这样一个点到根只有 \(\log n\) 条轻边，所以可以对每一条重链开一个树状数组维护，因为要动态加点，所以在链长大于数组长度是将数组长度扩倍，这样是均摊是 \(O(n)\) 的。
